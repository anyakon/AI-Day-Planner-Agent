# Governance

## Risk Register

| Риск | Вероятность | Влияние | Детект | Защита | Остаточный риск |
|-----|-------------|--------|--------|--------|----------------|
| Prompt injection | Средняя | Среднее | анализ входных данных | фильтрация запросов | низкий |
| Утечка персональных данных | Низкая | Высокое | аудит логов | анонимизация | низкий |
| Ошибки планирования | Средняя | Среднее | eval тесты | fallback алгоритм | средний |
| Ошибки API | Средняя | Низкое | мониторинг | retry | низкий |
| Неверные оценки времени | Высокая | Низкое | анализ данных | ручная корректировка | средний |

---

# Политика логов

Логи системы будут содержать:

- запросы пользователя
- действия агента
- вызовы инструментов
- ошибки

Персональные данные пользователя будут:

- минимизированы
- анонимизированы
- храниться ограниченное время.

---

# Работа с персональными данными

Система может обрабатывать:

- задачи пользователя
- данные календаря
- временные предпочтения

Меры защиты:

- не сохранять чувствительные данные
- использовать безопасное хранение
- ограничить доступ к логам

---

# Защита от Prompt Injection

Используются следующие меры:

1. разделение system prompt и пользовательского ввода
2. фильтрация опасных инструкций
3. ограничение доступа к инструментам
4. валидация вызовов инструментов

---

# Подтверждение действий

Критические действия требуют подтверждения пользователя:

- изменение расписания
- удаление задач
- интеграции с календарем

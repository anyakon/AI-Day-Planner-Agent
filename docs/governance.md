## Политика логов

Логи содержат:
- запросы пользователя (анонимизированные)
- вызовы инструментов (тип, аргументы без PII)
- ошибки выполнения
- метрики качества

Персональные данные:
- анонимизируются перед отправкой в LLM (security.py)
- не сохраняются в логах
- хранятся только в token.json для OAuth

MetricsCollector сохраняет в metrics.json:
- количественные метрики (runs, tasks, tokens)
- latency (avg, p50, p95)
- cost (USD)
- tool call counts
- error counts
- guardrail rejections

Langfuse отправляет трейсы в cloud:
- input/output каждого запуска
- metadata: response_time, iterations, tokens, cost

## Работа с персональными данными

Система обрабатывает:
- задачи пользователя (название, длительность, дедлайн, suggested_start)
- события календаря (название, время начала/конца)
- предпочтения (memory.json)

Меры защиты:
- anonymize_text() — замена PII на [REDACTED] через regex
- validate_tool_input() — блокировка PII в параметрах инструментов
- PII паттерны: телефоны, email, паспорта, карты, СНИЛС
- check_prompt_injection() — детект injection в именах задач

## Защита от Prompt Injection

- Разделение system prompt и пользовательского ввода
- 10 regex-паттернов для детекта injection (ignore instructions, system spoofing, bypass security, dangerous commands)
- Pydantic валидация с field_validator на имя задачи
- Валидация входных данных инструментов
- Rate limiting
- check_output_safety() — проверка ответа LLM на опасные паттерны

## Подтверждение действий

Критические действия:
- создание событий в календаре — требует предварительной валидации через validate_schedule
- валидация плана — автоматическая, без подтверждения
- удаление событий — не поддерживается
- guardrail rejection — автоматический, запрос отклоняется с ошибкой

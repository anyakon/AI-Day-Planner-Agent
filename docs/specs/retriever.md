# Retriever / Preferences

## Роль

Извлечение сохранённых предпочтений пользователя для учёта при планировании.

## Хранилище

Файл memory.json в корне проекта.

Формат:
```json
{
  "preferred_start_hour": 10,
  "lunch_time": "13:00-14:00"
}
```

## Инструменты

- save_preference(key, value) — сохранить предпочтение
- get_preferences() — прочитать все предпочтения

## Интеграция

Предпочтения доступны LLM через инструмент get_preferences. LLM решает учитывать их при составлении плана.

## Ограничения

- Нет векторного поиска
- Нет эмбеддингов
- Простое key-value хранилище

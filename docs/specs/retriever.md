# Retriever

- **Роль:** Извлечение правил планирования под конкретного пользователя.
- **Индекс:** Векторное хранилище (Faiss/ChromaDB) + метаданные (user_id).
- **Процесс:** 
  1. Пользователь пишет: "Запланируй тренировку".
  2. Embedder векторизует запрос.
  3. Поиск Top-3 релевантных фактов (например, "Тренировки только вечером", "Длительность тренировки обычно 1 час").
- **Ограничения (Agentic track):** Similarity threshold > 0.75. Если релевантных данных нет, полагаемся только на явный промпт пользователя, чтобы избежать подмешивания чужого контекста.

# Product Proposal
AI Day Planner Agent

---

# 1. Обоснование идеи

Планирование дня — одна из самых распространенных задач для людей с большим количеством обязанностей.

Несмотря на наличие календарей и таск-менеджеров, большинство пользователей:

- планируют задачи вручную
- переоценивают свои возможности
- не умеют эффективно расставлять приоритеты
- не адаптируют расписание при изменениях

AI-агент может автоматизировать этот процесс.

Агент способен:

- анализировать задачи
- учитывать дедлайны
- учитывать доступное время
- оптимизировать порядок задач
- перестраивать расписание

Это позволяет пользователю сосредоточиться на выполнении задач, а не на планировании.

---

# 2. Цель и метрики

## Цель проекта

Создать PoC агентной системы, которая автоматически строит и оптимизирует расписание дня пользователя.

---

## Продуктовые метрики

- время создания плана < 30 секунд
- план покрывает >80% задач пользователя
- пользователь принимает ≥70% предложенного плана

---

## Агентные метрики

- успешность планирования
- корректность приоритизации задач
- способность корректно реагировать на изменения

---

## Технические метрики

- p95 latency < 10 секунд
- успешность выполнения инструментов >95%

---

# 3. Потенциальные сценарии использования

## Сценарий 1 — планирование дня

Пользователь вводит:

- список задач
- дедлайны
- рабочие часы

Агент строит расписание.

---

## Сценарий 2 — изменение условий

Появляется новая задача.

Агент:

- пересчитывает план
- предлагает обновленное расписание.

---

## Сценарий 3 — конфликт задач

Две задачи имеют одинаковый дедлайн.

Агент:

- определяет приоритет
- предлагает альтернативные варианты.

---

## Edge cases

- слишком много задач
- задачи без оценки времени
- задачи с одинаковыми дедлайнами
- задачи больше доступного времени

---

# 4. Ограничения

## Технические ограничения

- p95 latency < 10 секунд
- лимиты LLM API
- ограниченное число внешних интеграций

---

## Операционные ограничения

- ограниченный бюджет на API
- PoC должен быть реализован за 1–2 недели
- команда из 2–3 человек

---

# 5. Архитектурный набросок

Система состоит из нескольких модулей:

### Planner Agent

LLM агент, который:

- анализирует задачи
- принимает решения
- вызывает инструменты

---

### Task Manager Tool

Инструмент для:

- хранения задач
- получения списка задач
- обновления задач

---

### Calendar Integration

Получает данные календаря пользователя.

---

### Schedule Optimizer

Алгоритм распределения задач по времени.

---

### Memory

Хранит:

- предпочтения пользователя
- прошлые планы

---

# 6. Потенциальный Data Flow

1. Пользователь вводит задачи
2. Агент получает календарь
3. Агент анализирует ограничения
4. Агент вызывает оптимизатор расписания
5. Генерируется план дня
6. План возвращается пользователю

---

## Что делает LLM

- анализирует задачи
- принимает решения
- определяет порядок задач
- объясняет результат

---

## Что НЕ делает LLM

- оптимизация расписания (алгоритм)
- хранение данных
- работа с API
